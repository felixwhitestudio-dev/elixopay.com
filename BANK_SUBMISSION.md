# 📄 เอกสารเสนอธนาคาร — Elixopay Merchant Acquiring Service

---

## 1. ภาพรวมธุรกิจ (Business Overview)

### Elixopay คืออะไร
Elixopay เป็น **ระบบ Payment Gateway** สำหรับร้านค้าออนไลน์ (Merchant Acquiring Service) ที่ช่วยให้ร้านค้า **รับชำระเงิน** จากลูกค้าผ่านช่องทางต่างๆ ได้แก่:

- **QR Code Payment** (PromptPay / Thai QR)
- **บัตรเครดิต/เดบิต** (Visa, Mastercard)
- **โอนเงินผ่านธนาคาร** (Bank Transfer)

### รูปแบบธุรกิจ
Elixopay ดำเนินธุรกิจในรูปแบบ **Non-Custodial Payment Facilitator** กล่าวคือ:

| ข้อ | รายละเอียด |
|---|---|
| ❌ **ไม่ถือเงินลูกค้า** | เงินจากลูกค้าจะเข้าบัญชีร้านค้าโดยตรงผ่าน Banking API |
| ❌ **ไม่ใช่ P2P** | ไม่มีการโอนเงินระหว่างผู้ใช้งาน |
| ❌ **ไม่เกี่ยวข้องกับ Cryptocurrency** | ไม่มีกระเป๋าเงินดิจิทัล ไม่มีการแลกเปลี่ยนสกุลเงินดิจิทัล |
| ✅ **เป็น Payment Gateway** | เชื่อมต่อร้านค้ากับ Banking API เพื่อรับชำระเงินจากลูกค้า |
| ✅ **คิดค่าธรรมเนียมต่อรายการ** | รายได้จากค่าธรรมเนียมการประมวลผลธุรกรรม (% ต่อรายการ) |

## 2. ขั้นตอนการชำระเงิน (Payment Flow)

```
ลูกค้า → เว็บไซต์ร้านค้า → Elixopay API → Banking API (KBank/SCB/BBL)
                                                   ↓
                                            บัญชีธนาคารร้านค้า
                                                   ↓
                                       ร้านค้าได้รับเงินโดยตรง
```

### ขั้นตอนโดยละเอียด

1. **ลูกค้า** เลือกสินค้าและกดชำระเงินบนเว็บไซต์ร้านค้า
2. **เว็บไซต์ร้านค้า** เรียก Elixopay API เพื่อสร้าง Payment Session
3. **Elixopay** สร้าง QR Code หรือ Checkout Page สำหรับลูกค้า
4. **ลูกค้า** ชำระเงินผ่าน QR Code / บัตรเครดิต / โอนเงิน
5. **Banking API** (KBank/SCB) ประมวลผลรายการและแจ้งกลับ
6. **Elixopay** ส่ง Webhook แจ้งร้านค้าว่าชำระเงินสำเร็จ
7. **เงินเข้าบัญชีร้านค้าโดยตรง** — Elixopay ไม่ถือเงินไว้

## 3. ความปลอดภัยและการปฏิบัติตามกฎหมาย (Security & Compliance)

### มาตรฐานความปลอดภัย

| มาตรฐาน | รายละเอียด |
|---|---|
| **PCI DSS Compliant** | เข้ารหัสข้อมูลบัตรเครดิตแบบ End-to-End ไม่เก็บหมายเลขบัตรบนเซิร์ฟเวอร์ |
| **SSL/TLS** | ทุก API endpoint ใช้ HTTPS พร้อม TLS 1.2+ |
| **API Key Authentication** | ร้านค้าต้องยืนยันตัวตนด้วย API Key ทุกครั้ง |
| **Webhook Signature** | ร้านค้าสามารถตรวจสอบลายเซ็นดิจิทัลของ webhook ได้ |
| **Rate Limiting** | จำกัดจำนวน request ต่อนาทีเพื่อป้องกัน DDoS |
| **Idempotency Keys** | ป้องกันการชำระเงินซ้ำจาก network timeout |

### การตรวจสอบตัวตน (KYC)

| ขั้นตอน | รายละเอียด |
|---|---|
| **การยืนยันอีเมล** | ร้านค้าต้องยืนยันอีเมลก่อนใช้งาน |
| **บัตรประชาชน/หนังสือเดินทาง** | อัปโหลดเอกสารยืนยันตัวตน |
| **Selfie พร้อมเอกสาร** | ถ่ายรูปคู่กับเอกสาร (ตามข้อกำหนด ธปท.) |
| **หนังสือรับรองบริษัท** | สำหรับนิติบุคคล |
| **การตรวจสอบโดย Admin** | ทีม compliance ตรวจสอบเอกสารทุกรายภายใน 1-2 วันทำการ |
| **บัญชีธนาคารล็อค** | หลัง KYC อนุมัติ ไม่สามารถเปลี่ยนบัญชีธนาคารได้เองต้องยื่นคำร้อง |

## 6. หลักฐานยืนยัน Non-Custodial Model

### สิ่งที่ระบบ **ไม่มี**

- ❌ ไม่มี **ฟังก์ชันถอนเงิน** (Withdraw) — เงินเข้าร้านค้าโดยตรง
- ❌ ไม่มี **ฟังก์ชันโอนเงินระหว่างผู้ใช้** (P2P Transfer)
- ❌ ไม่มี **กระเป๋าเงินดิจิทัล** (Crypto Wallet)
- ❌ ไม่มี **การแลกเปลี่ยนสกุลเงิน** (Exchange)
- ❌ ไม่มี **การเก็บรักษาเงินลูกค้า** (Custodial)

### สิ่งที่ระบบ **มี**

- ✅ **Payment Processing** — ประมวลผลรายการชำระเงิน
- ✅ **Settlement Tracking** — ติดตามยอดเงินที่ settle เข้าบัญชีร้านค้า
- ✅ **Refund Management** — จัดการการคืนเงิน (ผ่าน Banking API)
- ✅ **Fee Calculation** — คำนวณค่าธรรมเนียมอัตโนมัติ

## 8. คำขออนุญาต

Elixopay ขออนุญาตเชื่อมต่อ Banking API เพื่อให้บริการ Payment Gateway แก่ร้านค้าออนไลน์ โดยมีขอบเขตการใช้งานดังนี้:

1. **สร้าง QR Code Payment** สำหรับลูกค้าชำระเงิน
2. **รับ Webhook** แจ้งสถานะรายการจากธนาคาร
3. **ตรวจสอบสถานะรายการ** (Transaction Status Inquiry)

Elixopay **ไม่ได้ร้องขอ** สิทธิ์ในการ:
- โอนเงินออกจากบัญชี
- ถือเงินแทนบุคคลอื่น
- แลกเปลี่ยนสกุลเงินใดๆ
