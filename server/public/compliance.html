รูปแบบการให้บริการ (Business Model)

Elixopay เป็น ระบบรับชำระเงินสำหรับร้านค้า (Merchant Acquiring Service) ที่ช่วยให้ธุรกิจสามารถรับชำระเงินจากลูกค้าผ่านช่องทางออนไลน์ได้อย่างปลอดภัย

⚠️ Elixopay ไม่ได้ให้บริการโอนเงินระหว่างบุคคล (P2P Transfer) — เราให้บริการเฉพาะการรับชำระเงินจากลูกค้าของร้านค้า (Merchant) ที่ผ่านการตรวจสอบ KYC แล้วเท่านั้น

นโยบายป้องกันการฟอกเงิน (AML Policy)

Elixopay ดำเนินนโยบายป้องกันการฟอกเงินอย่างเข้มงวด ตามหลักการของ:

  • พ.ร.บ. ป้องกันและปราบปรามการฟอกเงิน พ.ศ. 2542 และฉบับแก้ไขเพิ่มเติม
  • พ.ร.บ. ป้องกันและปราบปรามการสนับสนุนทางการเงินแก่การก่อการร้าย พ.ศ. 2556
  • แนวทางของ FATF (Financial Action Task Force) ในการป้องกันการฟอกเงิน
  • ระเบียบของธนาคารแห่งประเทศไทย (BOT) ว่าด้วยการรู้จักลูกค้า

มาตรการที่ดำเนินการ

  • Customer Due Diligence (CDD) — ตรวจสอบตัวตนของ Merchant ทุกรายก่อนอนุมัติการใช้งาน
  • Enhanced Due Diligence (EDD) — ตรวจสอบเพิ่มเติมสำหรับ Merchant ที่มีความเสี่ยงสูง
  • Transaction Monitoring — ระบบ AI เฝ้าระวังธุรกรรมผิดปกติแบบ Real-time ตลอด 24 ชั่วโมง
  • Suspicious Transaction Reporting (STR) — รายงานธุรกรรมต้องสงสัยต่อ สำนักงาน ปปง. ตามกฎหมาย
  • Sanction Screening — ตรวจสอบรายชื่อบุคคลต้องห้ามจาก UN, OFAC, EU และ ปปง.

ข้อกำหนด KYC สำหรับ Merchant

Merchant ทุกรายที่สมัครใช้บริการ Elixopay จะต้องผ่านกระบวนการยืนยันตัวตน (KYC) ก่อนเปิดใช้งาน:

บุคคลธรรมดา (Individual Merchant)

  • สำเนาบัตรประชาชน / หนังสือเดินทาง
  • หลักฐานที่อยู่ (ไม่เกิน 3 เดือน)
  • ข้อมูลเว็บไซต์ / ช่องทางการขาย
  • รายละเอียดสินค้าหรือบริการที่จำหน่าย

นิติบุคคล (Corporate Merchant)

  • สำเนาหนังสือรับรองบริษัท (ไม่เกิน 3 เดือน)
  • สำเนาบัตรประชาชนของกรรมการผู้มีอำนาจลงนาม
  • ข้อมูล Beneficial Owner (ผู้รับประโยชน์ที่แท้จริง) ที่ถือหุ้น 25% ขึ้นไป
  • หลักฐานที่อยู่สำนักงาน
  • ใบอนุญาตประกอบธุรกิจ (ถ้ามี)

ประเภทธุรกิจที่ไม่ให้บริการ (Prohibited Business)

ระบบเฝ้าระวังธุรกรรม (Transaction Monitoring)

Elixopay ใช้ระบบ AI-Powered Transaction Monitoring เพื่อตรวจจับกิจกรรมผิดปกติ:

  • Real-time Fraud Detection — ตรวจจับรายการต้องสงสัยทันทีที่เกิดขึ้น
  • Velocity Check — ตรวจสอบความถี่ของธุรกรรมที่ผิดปกติ
  • Amount Threshold Alert — แจ้งเตือนเมื่อยอดเงินเกินเกณฑ์ที่กำหนด
  • Pattern Analysis — วิเคราะห์รูปแบบธุรกรรมเพื่อตรวจจับ Structuring
  • Auto-Suspension — ระงับบัญชีอัตโนมัติเมื่อพบกิจกรรมต้องสงสัย

มาตรฐานความปลอดภัย

  • PCI-DSS Level 1 — มาตรฐานความปลอดภัยสูงสุดสำหรับการประมวลผลบัตรเครดิต
  • TLS 1.3 Encryption — เข้ารหัสข้อมูลทุกการสื่อสาร
  • Tokenization — ไม่เก็บข้อมูลบัตรเครดิตโดยตรง
  • 3D Secure 2.0 — ยืนยันตัวตนผู้ชำระเงินเพิ่มเติม
  • SOC 2 Type II — ตรวจสอบระบบควบคุมภายในอย่างต่อเนื่อง
